Token Generate

Normal

POST /marketlive/auth/token

Header Authorization: Basic base64(ConsumerKey:ConsumerSecret)

cURL curl --request POST 'https://amitofile.com/marketlive/auth/token' --header 'Authorization: Basic base64(ConsumerKey:ConsumerSecret)'

Description This is a regular API to generate access token for websocket connection.
Required: Authorization credentials.

With application(s)

POST /marketlive/auth/token

Header Authorization: Basic base64(ConsumerKey:ConsumerSecret)
Content-Type: application/x-www-form-urlencoded

Body application={your-application}

cURL curl --request POST 'https://amitofile.com/marketlive/auth/token' --header 'Authorization: Basic base64(ConsumerKey:ConsumerSecret)' --header 'Content-Type: application/x-www-form-urlencoded' --data-urlencode 'application=your-application'

Description Access tokens are application based. If application name is not provided, "default" application is selected for generating access tokens.
Required: Authorization credentials.

Note: User can provide only registered application name.

With scope

POST /marketlive/auth/token

Header Authorization: Basic base64(ConsumerKey:ConsumerSecret)
Content-Type: application/x-www-form-urlencoded

Body scope={prices|orders|all}

cURL curl --request POST 'https://amitofile.com/marketlive/auth/token' --header 'Authorization: Basic base64(ConsumerKey:ConsumerSecret)' --header 'Content-Type: application/x-www-form-urlencoded' --data-urlencode 'scope=prices|orders|all'

Description Scopes are used to limit usability of access tokens. For example, access token generated using "prices" scope can be used to connect price feed only. Likewise "orders" scope for order feed and "all" scope for any feed.
Required: Authorization credentials.

Token regenerate

POST /marketlive/auth/token/regenerate

Header Authorization: Basic base64(ConsumerKey:ConsumerSecret)

cURL curl --request POST 'https://amitofile.com/marketlive/auth/token/regenerate' --header 'Authorization: Basic base64(ConsumerKey:ConsumerSecret)'

Description User can regenerate access token for specific application or scope. New access token will be generated if no old access token found for given specification.
Required: Authorization credentials.

Note: Old access token with same specification will be disabled after regenerating new.

Events

broadcast This event relays the information related to websocket connection. Every websocket should listen to this event throughout.
socket.on("broadcast", function (msg) => {
   console.log(msg);
});

ping This event is very important. For keeping websocket connected user must emit socket id in response to ping event.
Any connection failing to respond ping event will be terminated.
socket.on("ping", () => {
   socket.emit("ping", socket.id);
});

getstock Capture stock price information.
socket.on("getstock", function (data) => {
   console.log(data);
});

getmarket Capture stock market updates.
socket.on("getmarket", function (data) => {
   console.log(data);
});

getindice Capture indice information.
socket.on("getindice", function (data) => {
   console.log(data);
});

getexchange Capture exchange status updates.
socket.on("getexchange", function (data) => {
   console.log(data);
});

getorderstatus Capture order status information (for your orders).
socket.on("getorderstatus", function (data) => {
   console.log(data);
});

stockload Load script(s) for getstock event in price feed. Only 15 wtokens can be passed in single emit.
socket.emit("stockload", { "wtokens": "wtoken1,wtoken2,...,wtoken15" });
 or
socket.emit("stockload", { "wtokens": [wtoken1,wtoken2,...,wtoken15] });
 or
socket.emit("stockload", "wtoken1,wtoken2,...,wtoken15");
 or
socket.emit("stockload", [wtoken1,wtoken2,...,wtoken15]);

stockunload Remove stock(s) from getstock event. Only 15 wtokens can be passed in single emit.
socket.emit("stockunload", { "wtokens": "wtoken1,wtoken2,...,wtoken15" });
 or
socket.emit("stockunload", { "wtokens": [wtoken1,wtoken2,...,wtoken15] });
 or
socket.emit("stockunload", "wtoken1,wtoken2,...,wtoken15");
 or
socket.emit("stockunload", [wtoken1,wtoken2,...,wtoken15]);

marketload Load script(s) for getmarket event in price feed.
socket.emit("marketload", { "wtokens": "wtoken1,wtoken2,...,wtoken15" });

marketunload Remove stock(s) from getmarket event.
socket.emit("marketunload", { "wtokens": "wtoken1,wtoken2,...,wtoken15" });

indiceload Load script(s) for getindice event in price feed.
socket.emit("indiceload", { "wtokens": "wtoken1,wtoken2,...,wtoken15" });

indiceunload Remove script(s) from getindice event.
socket.emit("indiceunload", { "wtokens": "wtoken1,wtoken2,...,wtoken15" });

exchangeload Load exchange codes (NN or BN) for getexchange.
socket.emit("exchangeload", "NN");
 or
socket.emit("exchangeload", "BN");

exchangeunload Remove exchange code from getexchange.
socket.emit("exchangeunload", "NN");
 or
socket.emit("exchangeunload", "BN");

json Convert feed response in JSON or non jSON (CSV) format.
socket.emit("json", "true");
 or
socket.emit("json", "false");

leaveall Deselect all scripts. Feed will stop but not disconnect, user can load new scripts.
socket.emit("leaveall", "true");
 or
socket.emit("leaveall", "false");
